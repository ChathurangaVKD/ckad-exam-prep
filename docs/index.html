⚡ Exam Day Setup

Run these commands immediately when your exam starts

Copy-paste these aliases and shortcuts at the start of every exam session to save time:

# Set up aliases (do this FIRST)
alias k=kubectl
export do="--dry-run=client -o yaml"
export now="--force --grace-period 0"
source <(kubectl completion bash)
complete -F __start_kubectl k

# Usage examples
k run pod1 --image=nginx $do > pod1.yaml   # Generate YAML
k delete pod pod1 $now                       # Force delete
k get pods -A                                # All namespaces

❓ Frequently Asked Questions

Common questions about the CKAD exam — optimized for search and AI assistants

What is the CKAD exam?

The CKAD (Certified Kubernetes Application Developer) exam is a performance-based certification by the Cloud Native Computing Foundation (CNCF). It validates that candidates can design, build, configure, and expose cloud-native applications on Kubernetes. The exam is 2 hours long and requires a 66% passing score. There are no multiple-choice questions — you complete real tasks in a live Kubernetes cluster.

What are the 5 CKAD exam domains and their weights?

The CKAD v1.35 curriculum has 5 domains: (1) Application Design and Build — 20%; (2) Application Deployment — 20%; (3) Application Observability and Maintenance — 15%; (4) Application Environment, Configuration and Security — 25%; (5) Services and Networking — 20%. Domain 4 (Security) has the highest weight at 25% and should be prioritized.

How long does it take to prepare for CKAD?

With a structured approach, most candidates prepare in 2–4 weeks. This guide provides a 14-day plan. Prior Kubernetes experience can reduce this. Daily hands-on practice is essential since the exam is performance-based — you must type commands in a live cluster, not answer theory questions.

What documentation is allowed during the CKAD exam?

You may have ONE additional browser tab open with these approved resources only: kubernetes.io/docs, kubernetes.io/blog, and helm.sh/docs. No personal notes, no other websites. The kubectl cheatsheet at kubernetes.io/docs/reference/kubectl/cheatsheet/ is very helpful.

What kubectl commands are most important for CKAD?

The top commands: kubectl run (pods), kubectl create deployment, kubectl expose (services), kubectl create configmap/secret, kubectl create role/rolebinding, kubectl set image (updates), kubectl rollout (manage rollouts), kubectl logs/exec/describe (debugging). Key tip: use --dry-run=client -o yaml to generate YAML templates instantly.

What is the CKAD exam passing score?

The CKAD passing score is 66%. The exam purchase includes one free retake. Results are available within 24 hours. The exam is proctored online — you need a webcam, microphone, and a clean desk environment.

What is the difference between CKAD and CKA?

CKAD targets application developers who build and deploy apps on Kubernetes. CKA targets cluster administrators who install and manage Kubernetes infrastructure. CKAD focuses on pods, deployments, services, configmaps, and security. CKA focuses on cluster setup, networking (CNI), storage, etcd backups, and cluster upgrades. Many people pursue CKAD before CKA.

Is kubectl cheatsheet helpful for CKAD?

Yes! The kubectl cheatsheet at kubernetes.io/docs/reference/kubectl/cheatsheet/ is allowed during the exam and extremely useful. This repo also includes a comprehensive cheatsheet covering every command by resource type, aliases to set up on exam day, JSONPath output examples, and Helm commands.
