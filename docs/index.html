Exam Day

First Thing You Do

Copy-paste these aliases at the start of your exam session. They save 30-40% of your time.

bash — exam day setup 
# ⚡ Run these IMMEDIATELY when your exam starts
alias k=kubectl
export do="--dry-run=client -o yaml"
export now="--force --grace-period 0"
source <(kubectl completion bash)
complete -F __start_kubectl k

# Usage — generate YAML in 2 seconds:
k run pod1 --image=nginx $do > pod.yaml

# Force delete a stuck pod:
k delete pod pod1 $now

# Switch context (each question specifies one):
kubectl config use-context my-cluster

FAQ

Frequently Asked Questions

Common questions about the CKAD exam answered clearly and concisely.

What is the CKAD exam and who is it for?
The CKAD (Certified Kubernetes Application Developer) is a hands-on, performance-based certification by the Cloud Native Computing Foundation (CNCF). You complete real tasks in a live Kubernetes cluster — no multiple choice. It's designed for developers who build and deploy applications on Kubernetes. Passing score: 66%. Duration: 2 hours.
What are the 5 CKAD exam domains and their weights?
The CKAD v1.35 curriculum covers 5 domains:

1. Application Design and Build — 20% (containers, jobs, multi-container pods, volumes)
2. Application Deployment — 20% (deployments, Helm, Kustomize, DaemonSets)
3. Observability and Maintenance — 15% (probes, logging, debugging)
4. Environment, Configuration and Security — 25% ← highest weight!
5. Services and Networking — 20% (services, ingress, network policies)
What kubectl aliases should I set up on exam day?
Run these immediately when your exam starts: 
alias k=kubectl
export do="--dry-run=client -o yaml"
export now="--force --grace-period 0"
source <(kubectl completion bash)
complete -F __start_kubectl k
Then use: k run pod1 --image=nginx $do > pod.yaml to generate YAML in 2 seconds.
What documentation is allowed during the CKAD exam?
You may have one additional browser tab open during the exam with only these approved sites:
• kubernetes.io/docs
• kubernetes.io/blog
• helm.sh/docs

No personal notes or other websites. Bookmark the kubectl cheatsheet at kubernetes.io/docs/reference/kubectl/cheatsheet/ before your exam.
How long does it take to prepare for CKAD?
Most candidates prepare in 2–4 weeks with daily hands-on practice. This guide includes a structured 14-day study plan. Prior Kubernetes experience can shorten this significantly. The key is daily hands-on practice — the exam is performance-based, so typing commands is essential.
What is the difference between CKAD and CKA?
CKAD is for application developers who build and deploy apps on Kubernetes (pods, deployments, services, configmaps, security). CKA is for cluster administrators who manage Kubernetes infrastructure (cluster setup, etcd, networking plugins, upgrades). Many take CKAD before CKA as a stepping stone.
How do I quickly create Kubernetes resources on exam day?
Use imperative commands instead of writing YAML from scratch: 
kubectl run nginx --image=nginx:1.25                          # Pod
kubectl create deployment myapp --image=nginx --replicas=3    # Deployment
kubectl expose deployment myapp --port=80 --type=ClusterIP    # Service
kubectl create configmap my-cm --from-literal=KEY=VALUE       # ConfigMap
kubectl create secret generic sec --from-literal=pass=secret  # Secret
kubectl create role my-role --verb=get,list --resource=pods   # Role
kubectl create rolebinding rb --role=my-role --sa=default:sa  # RoleBinding
